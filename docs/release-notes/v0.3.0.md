## Release v0.3.0

### EmDash 0.9.x support

- **`@emplugins/emdash-plugin-md-draft`**
  - Peer dependency: **`emdash` `>=0.9.0`** (no upper bound).
  - Plugin capability updated from deprecated **`write:content`** to **`content:write`** (required for `emdash plugin publish`).
  - Ingest no longer collapses `ctx.content.create` failures to a single generic message: EmDash structured codes (**`VALIDATION_ERROR`**, **`SLUG_CONFLICT`**, **`CONFLICT`**, **`NOT_FOUND`**) are forwarded as `PluginRouteError` with the same code and HTTP status where applicable; unknown failures use **`CREATE_FAILED`** (400).

- **`@emplugins/mcp-emdash-drafts`**
  - Documentation-only alignment: targets sites running the plugin on EmDash **0.9.x+** (tooling and env vars unchanged).

### Migration

- Upgrade your site’s **`emdash`** package to **0.9.0** or newer, then bump **`@emplugins/emdash-plugin-md-draft`**.
- No MCP client env changes; ingest URL and HMAC contract are unchanged.
