## Release v0.4.0

### EmDash 0.14.x support

- **`@emplugins/emdash-plugin-md-draft`**
  - Peer dependency: **`emdash` `>=0.14.0`** (raised from `>=0.9.0`).
  - Optional YAML frontmatter **`locale`** and **`translationOf`** for multilingual sites; slug checks are scoped per locale when `locale` is set.
  - Ingest success response includes **`locale`** when provided.

- **`@emplugins/mcp-emdash-drafts`**
  - Tool descriptions and `validate_markdown` output mention locale / translation fields.
  - Targets sites on EmDash **0.14.x+** (no env or HMAC contract changes).

- **`@emplugins/shared`**
  - Frontmatter schema accepts `locale` / `translationOf` with validation (`translationOf` requires `locale`).

### Migration

1. Upgrade your site’s **`emdash`** package to **0.14.0** or newer (sites on 0.9–0.13 must upgrade EmDash before EmPost).
2. Bump **`@emplugins/emdash-plugin-md-draft`** to v0.4.x.
3. MCP clients: no env changes unless rotating secrets.
4. Multi-locale sites: optionally add `locale` (and `translationOf` for translations) to markdown frontmatter.

### Unchanged

- Ingest URL, HMAC signing, JSON body `{ "markdown": "..." }`, plugin id `empost-md-draft`, `content:write` capability.

### Operator notes

- EmDash **0.11+** enforces image/file MIME rules on save; invalid `seo.image` values may return `VALIDATION_ERROR`.
- Pre-publish smoke: health, default ingest, locale ingest (if i18n enabled), MCP tools — see [release checklist](../release-checklist.md).
